# 5. Business Requirements

## 5.1 Introduction

This chapter defines the high-level business requirements for the Restaurant Booking Management System (RBMS). These requirements describe the capabilities the system must provide to meet the business objectives identified in the previous chapters.

---

## 5.2 Customer Requirements

The system shall allow customers to:

* Create a customer account.
* Log in securely.
* View restaurant information and opening hours.
* Search for available reservation time slots.
* Create a reservation.
* View upcoming and past reservations.
* Modify a future reservation.
* Cancel a future reservation within the allowed cancellation period.
* Manage personal profile information.

---

## 5.3 Reservation Requirements

The system shall:

* Check table availability before confirming a reservation.
* Prevent double booking of tables.
* Assign an appropriate table based on party size.
* Store reservation date and time.
* Store the number of guests.
* Record reservation status.
* Keep a complete reservation history.
* Allow staff to manually create reservations for walk-in customers.

---

## 5.4 Table Management Requirements

The system shall allow authorized staff to:

* Create new tables.
* Update table information.
* Remove inactive tables.
* Define table capacity.
* Change table status.
* View all tables and their current availability.

---

## 5.5 Staff Requirements

Authorized staff shall be able to:

* View today's reservations.
* Search reservations by customer name, date, or status.
* Confirm reservations.
* Check in arriving customers.
* Mark reservations as completed.
* Mark customers as no-shows.
* Cancel reservations when necessary.

---

## 5.6 Administration Requirements

The system shall allow administrators to:

* Manage customer accounts.
* Manage staff accounts.
* Configure restaurant opening hours.
* Configure holiday closures.
* View dashboard statistics.
* Access system reports.

---

## 5.7 Reporting Requirements

The system shall generate reports including:

* Daily reservations.
* Weekly reservations.
* Monthly reservations.
* Table utilization.
* Reservation status summary.
* Customer activity summary.

---

## 5.8 Security Requirements

The system shall:

* Require authentication before accessing protected features.
* Restrict access based on user roles.
* Encrypt user passwords.
* Validate all user input.
* Record important system activities through audit logs.

---

## 5.9 Summary

These business requirements define the core functionality required for Version 1.0 of the Restaurant Booking Management System. Detailed functional and non-functional requirements will be specified in the Software Requirements Specification (SRS).
