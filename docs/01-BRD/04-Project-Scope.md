# 4. Project Scope

## 4.1 Purpose

This chapter defines the boundaries of Version 1.0 of the Restaurant Booking Management System (RBMS). It specifies the functionality that will be delivered in the initial release and identifies features that are intentionally excluded.

---

## 4.2 In Scope

The first version of the system will include the following capabilities:

### Customer Module

* Customer registration
* Secure login
* Profile management
* Table reservation
* Reservation history
* Reservation modification
* Reservation cancellation

### Reservation Module

* Reservation scheduling
* Reservation validation
* Reservation status management
* Reservation search

### Table Management Module

* Create tables
* Update tables
* Delete tables
* Manage table capacity
* Track table status

### Administration Module

* Dashboard
* Customer management
* Reservation management
* Restaurant settings
* Opening hours
* Holiday management

### Reporting Module

* Daily reservation statistics
* Table utilization summary
* Customer statistics

---

## 4.3 Out of Scope

The following features are not included in Version 1.0:

* Online food ordering
* Online payments
* Delivery management
* Mobile applications
* Multi-branch support
* Loyalty programs
* Employee scheduling
* QR code ordering

These features may be considered in future releases.

---

## 4.4 Project Assumptions

The following assumptions apply:

* The restaurant operates from a single location.
* Customers have internet access.
* Restaurant staff receive basic system training.
* Reservations are managed in English.

---

## 4.5 Project Constraints

The project is subject to the following constraints:

* Single restaurant only.
* Web application only.
* No external payment gateway.
* No third-party reservation platforms.

---

## 4.6 Summary

The project scope ensures that Version 1.0 remains focused on delivering a complete reservation management solution while leaving advanced functionality for future releases.
