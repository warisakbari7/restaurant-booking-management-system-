# 3. Stakeholder Analysis

## 3.1 Purpose

This chapter identifies the individuals and groups who interact with the Restaurant Booking Management System (RBMS). Understanding stakeholder responsibilities helps ensure that the system supports real business needs and provides appropriate functionality for each type of user.

---

## 3.2 Primary Stakeholders

| Stakeholder          | Description                                                    |
| -------------------- | -------------------------------------------------------------- |
| Restaurant Owner     | Owns the business and reviews operational performance.         |
| Restaurant Manager   | Manages daily restaurant operations and oversees reservations. |
| Receptionist / Host  | Handles customer reservations and table assignments.           |
| Customer             | Creates, modifies, and cancels reservations.                   |
| System Administrator | Configures the system and manages user accounts.               |

---

## 3.3 Stakeholder Responsibilities

### Restaurant Owner

The Restaurant Owner is responsible for monitoring business performance and ensuring that restaurant operations meet business objectives.

Expected system capabilities:

* View reservation statistics.
* Monitor table utilization.
* Review operational reports.

---

### Restaurant Manager

The Restaurant Manager supervises daily operations and ensures reservations are handled efficiently.

Expected system capabilities:

* View all reservations.
* Manage reservation schedules.
* Monitor table availability.
* Access operational dashboards.

---

### Receptionist / Host

The Receptionist or Host interacts with customers and manages reservations during business hours.

Expected system capabilities:

* Create reservations.
* Modify reservations.
* Cancel reservations.
* Assign tables.
* Search customer bookings.

---

### Customer

Customers use the system to reserve tables online.

Expected system capabilities:

* Register an account.
* Log in securely.
* Search available reservation times.
* Create reservations.
* Modify reservations.
* Cancel reservations.
* View reservation history.

---

### System Administrator

The System Administrator maintains the application configuration.

Expected system capabilities:

* Manage user accounts.
* Configure restaurant settings.
* Manage opening hours.
* View activity logs.

---

## 3.4 Stakeholder Influence

| Stakeholder          | Influence | System Usage |
| -------------------- | --------- | ------------ |
| Restaurant Owner     | High      | Medium       |
| Restaurant Manager   | High      | High         |
| Receptionist / Host  | Medium    | High         |
| Customer             | Medium    | High         |
| System Administrator | High      | Medium       |

---

## 3.5 Summary

The stakeholder analysis identifies the primary users of the Restaurant Booking Management System and their responsibilities. These stakeholder roles will be used later to define user permissions, functional requirements, and application modules in the Software Requirements Specification (SRS).
