# 7. Risks and Assumptions

## 7.1 Introduction

This chapter identifies the assumptions made during the planning of the Restaurant Booking Management System (RBMS) and highlights potential risks that could affect the successful delivery and operation of the project.

---

## 7.2 Project Assumptions

The project is based on the following assumptions:

* The restaurant operates from a single location.
* Restaurant staff have basic computer skills.
* Customers have internet access to make online reservations.
* The restaurant has a stable internet connection.
* Reservation information is stored electronically.
* Restaurant operating hours are maintained by administrators.
* The application will initially support the English language only.

---

## 7.3 Project Risks

| Risk                       | Impact | Mitigation                                               |
| -------------------------- | ------ | -------------------------------------------------------- |
| Incorrect reservation data | High   | Validate all user input before saving                    |
| Double booking             | High   | Verify table availability before confirming reservations |
| Unauthorized system access | High   | Implement authentication and role-based authorization    |
| Server downtime            | Medium | Use regular backups and reliable hosting                 |
| Data loss                  | High   | Perform scheduled database backups                       |
| User input errors          | Medium | Provide clear validation messages                        |
| Future feature expansion   | Low    | Design a modular and scalable architecture               |

---

## 7.4 Summary

The identified assumptions and risks provide guidance during development and help ensure that potential issues are considered early in the project lifecycle.
