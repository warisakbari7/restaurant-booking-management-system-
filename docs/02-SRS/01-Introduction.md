# 1. Introduction

## 1.1 Purpose

The purpose of this Software Requirements Specification (SRS) is to define the functional and non-functional requirements for Version 1.0 of the Restaurant Booking Management System (RBMS).

This document provides a complete technical description of the system and serves as the primary reference for software design, development, testing, and future maintenance.

---

## 1.2 Project Scope

The Restaurant Booking Management System is a web-based application that enables customers to reserve restaurant tables online while allowing restaurant staff to manage reservations, tables, customers, and restaurant settings through an administrative dashboard.

Version 1.0 focuses on providing a reliable, secure, and user-friendly reservation management solution for a single restaurant.

## 1.4 Definitions

| Term          | Definition                                                    |
| ------------- | ------------------------------------------------------------- |
| RBMS          | Restaurant Booking Management System                          |
| Customer      | A registered user who creates reservations                    |
| Reservation   | A booking made for a table at a specific date and time        |
| Table         | A physical dining table inside the restaurant                 |
| Staff         | Restaurant employees who manage reservations                  |
| Administrator | User responsible for system configuration and user management |
