## Purpose

This Software Requirements Specification (SRS) defines the functional and non-functional requirements for Version 1.0 of the Restaurant Booking Management System (RBMS). It serves as the primary technical reference for designing, implementing, testing, and maintaining the application.

The document is intended for developers, testers, designers, and future contributors. Every requirement described in this document will be implemented in the project and traced to the database, backend services, frontend components, and API endpoints.
